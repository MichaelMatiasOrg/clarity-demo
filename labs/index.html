Before you begin, you'll need a few things installed on your machine. Don't worry — each one is a quick setup, and you only do this once.

Install Git (the save system)

Git is like a "save game" system for code. Every time we make changes, Git remembers what changed and lets us undo mistakes. GitHub is the website where our project lives online — think of it as Google Drive, but for code.

Already have Git? Type git --version in your terminal. If you see a version number, skip to "Create a GitHub Account" below.

Mac: Open Terminal (search "Terminal" in Spotlight) and run:

xcode-select --install

A popup will appear — click "Install" and wait for it to finish. That's it.

Windows: Download Git from git-scm.com and run the installer. Click "Next" through all the screens — the defaults are perfect.

Create a GitHub Account

If you don't have one yet, go to github.com/join and sign up. It's free. Use your work email so your team lead can give you access to our project.

Connect Git to GitHub

Tell Git who you are (use the name and email from your GitHub account):

git config --global user.name "Your Name"
git config --global user.email "your@email.com"

Then log in to GitHub from your terminal — this is a one-time setup:

gh auth login
Don't have the gh command? Install it first: Mac: brew install gh — Windows: download from cli.github.com. If you don't have brew either, install it from brew.sh (just paste the one-liner they show you into Terminal).

Follow the prompts — pick "GitHub.com", "HTTPS", and "Login with a web browser". It will open your browser to confirm. Done.

Checkpoint: Run git --version — you should see something like git version 2.x.x. If yes, you're good to go.

Install Claude Code

Claude Code is Anthropic's CLI tool that lets you work with Claude directly in your terminal.

npm install -g @anthropic-ai/claude-code
You'll need Node.js 18+ installed first. Run node --version to check.

Install the Superpowers Plugin

This plugin gives Claude Code extra skills for building presentations — brainstorming, frontend design, and structured workflows.

claude install @anthropic-ai/claude-code-superpowers

Get Repository Access

Ask a team lead for access to the MichaelMatiasOrg/clarity-demo GitHub repository. You'll need push permissions to contribute decks.

Verify everything works: run claude --version in your terminal. You should see a version number.

Clone the repository and open it locally to make sure everything loads correctly.

git clone https://github.com/MichaelMatiasOrg/clarity-demo.git
cd clarity-demo

Verify It Works

Open index.html in your browser. You should see the Clarity hub page with all presentations listed.

open index.html
If you see the Clarity logo and presentation cards, you're all set. No build step needed — it's all static HTML.

You don't need to memorize all this — Claude reads the project files automatically. But here's a quick mental model of how things are organized.

CLAUDE.md is the Brain

The CLAUDE.md file at the root is the project's instruction manual. When Claude Code opens the repo, it reads this file first. It contains the design system, color palette, file structure, and all the rules Claude follows.

Shared Assets

The shared/ folder contains everything presentations have in common:

  • CSS — design system, animations, narration styles
  • JS — navigation, narration engine, dev mode
  • Logos — 25+ integration partner logos (SVG/PNG)
  • Screenshots — product UI images

The Template System

Every new presentation starts from presentations/_template/. It has the boilerplate HTML, CSS, and narration data structure ready to go. Claude knows to use this as a starting point.

Slide Structure

Each slide is a <section class="slide"> element. Slides snap to full screen and are navigated with arrow keys. Each gets a unique data-slide-id attribute.

You don't need to understand the code — Claude handles all of this. Just know that the structure exists so you can speak intelligently about it.

This is the fun part. Launch Claude Code from the repo root and describe what you want to build.

claude

Describe Your Deck

Once Claude is running, describe the presentation you want to create. Be specific about the audience, the story, and any integrations to highlight. Here are some example prompts:

Build a 10-slide deck for HR leaders at companies
using Lever, showing how Clarity catches identity
fraud during the hiring process. Include a
real-world attack story and the Lever integration.
Create a short 6-slide overview for a Zoom call
with Okta's security team. Focus on how Clarity
integrates with Okta for continuous employee
identity verification.
Claude will start with a brainstorming phase — asking you clarifying questions and proposing a slide outline before writing any code. Collaborate on the outline until you're happy.

What Makes a Good Prompt

  • Target audience — who will see this deck?
  • Integration focus — which ATS or platform? (Greenhouse, Lever, Okta...)
  • Slide count — how long should it be?
  • Key message — what should the viewer walk away remembering?

Once you approve the outline, Claude follows a structured process. Here's what happens behind the scenes so you know what to expect.

1. Feature Branch

Claude creates a new git branch (like add-lever-hr-deck) to keep your work isolated from the main codebase. This is a safety measure — nothing touches production until you're ready.

2. Scaffold the Presentation

Claude copies the _template/ folder into a new directory under presentations/ and sets up the file structure:

  • index.html — the presentation itself
  • presentation.css — slide-specific styles
  • narration-data.js — narration text and timing

3. Build the Slides

Claude writes each slide as a <section> element, using the shared design system for consistent typography, colors, and layout. It pulls in real logos from shared/logos/ and product screenshots.

4. Add Slide-Specific CSS

Custom animations, layouts, and visual effects go into the presentation's own CSS file — keeping things modular.

5. Update the Hub

Claude adds a card for your new deck on the hub landing page (index.html) so the team can find it.

Throughout this process, Claude will show you what it's doing and ask for approval before making changes. You're always in control.

Before your deck goes live, it needs to pass the project's automated tests. Claude handles this workflow too.

Run Tests Locally

The project uses Playwright for smoke tests. These verify that all pages load, links work, and there are no console errors.

npx playwright test

The Git Workflow

Claude follows a specific branching workflow to keep things safe:

  1. All work happens on the feature branch (never directly on main)
  2. Before pushing, Claude pulls the latest main and merges locally
  3. A pre-push hook automatically runs Playwright tests — the push only succeeds if all tests pass
  4. Once pushed, the changes deploy automatically via GitHub Pages
If tests fail, don't panic. Claude will read the error output and fix the issue automatically. Just let it work through the problem.
