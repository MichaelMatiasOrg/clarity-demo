Design Rules

Do

  • Use flat colors (no gradients on brand elements)
  • Maintain high contrast (white text on navy)
  • Use official logo SVG files only
  • Keep plenty of whitespace
  • Use Manrope font consistently
  • One key message per slide
  • Cite sources for all statistics
  • Use one-line titles (no wrapping)
  • Show comparisons visually (strikethrough old values)
  • Subtle animations only (rotating rings, pulsing elements)

Do Not

  • Use teal (#00d4aa) instead of Clarity Green
  • Use Inter or other substitute fonts
  • Add effects (shadows, glows) to the logo
  • Stretch or distort the logo
  • Use emojis in presentations
  • Create fake or invented SVG logos
  • Use busy or cluttered layouts
  • Use flashy or dramatic animations
  • Use gradients on the green accent color
  • Use dark theme unless specifically appropriate

Slide Types

Hero Slides

  • Large headline, minimal text
  • Subtle geometric background accents
  • Green accent on key words

Content Slides

  • Clear hierarchy with cards/boxes
  • Custom SVG icons, not stock
  • One key message per slide

Data / Stats Slides

  • Large numbers with context
  • Citations for credibility
  • Comparisons: old → new

Mockup Slides

  • Pixel-perfect, photorealistic
  • Realistic data, not placeholder text
  • Match real platform UIs (LinkedIn, Zoom)

Animation Guidelines

Animations should enhance comprehension, not attract attention to themselves.

Principles

Subtle
Enhance, not distract
Purposeful
Communicate something
Smooth
Ease-in-out timing
Restrained
No flashy effects

Recommended Types

Use

  • Scanning/sweep effects for detection features
  • Gentle pulse for live indicators
  • Flow animations for process diagrams
  • Fade transitions between states
  • Rotating rings for continuous processes
  • Staggered entrance for lists/cards

Avoid

  • Bouncing or shaking effects
  • Fast or erratic movement
  • Multiple competing animations
  • Animations without purpose
  • Particle effects or confetti (except landing)
  • 3D transforms or perspective shifts

Imagery & Mockups

Do

  • Use pixel-perfect, photorealistic mockups
  • Show realistic data (not "Lorem ipsum")
  • Include subtle UI details (status bars, proper spacing)
  • Match the Clarity color palette
  • Match real platform UIs (LinkedIn, Zoom, etc.)
  • Use proper fonts and spacing from real platforms

Do Not

  • Use obviously fake or placeholder content
  • Show buggy or incomplete UI
  • Mix different design systems in one mockup
  • Invent company logos — fetch real ones
  • Use low-resolution screenshots
  • Include unpolished or draft-quality mockups
